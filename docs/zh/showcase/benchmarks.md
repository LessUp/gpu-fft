## 正确阅读这页的方法

- 这些数字只代表 **CPU 路径**，因为当前 CI 环境没有 WebGPU adapter。
- benchmark 的原则是：**任何环境都输出 CPU 实测结果**，**只有检测到 WebGPU 时才输出 GPU 实测结果**。
- 小尺寸的波动更多受 JavaScript runtime 开销影响，而不只是 FFT 算法本身。
- GPU crossover point 必须被视作硬件相关问题，而不是仓库级固定营销口径。

## 在你的硬件上复现

```bash
npm run benchmark
```

输出会包含：

1. 每个测试尺寸的 CPU 耗时
2. 仅当 WebGPU 实际可用时出现的 GPU 耗时
3. 用于解释结果的环境信息

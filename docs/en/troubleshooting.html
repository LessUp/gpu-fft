Troubleshooting

Common issues you might encounter and how to resolve them.

WebGPU Not Available

FFTError: WebGPU is not available

This error occurs when WebGPU is not supported by the browser or has been disabled.

Solutions:
  • Check browser compatibility in Browser Compatibility
  • Update browser to latest version (Chrome 113+, Edge 113+, Firefox 128+)
  • Use CPU fallback: cpuFFT() / cpuIFFT()

Invalid Input Size

FFTError: Input size must be a power of 2

FFT requires input length to be a power of 2 (2, 4, 8, 16, ..., 65536).

Solutions:
  • Pad your data to next power of 2
  • Use validateFFTInput() before calling FFT
  • Check input array length (must be 2×N for N complex numbers)

Input Too Large

FFTError: Input size exceeds maximum of 65536

The maximum supported input size is 65,536 complex numbers.

Solutions:
  • Split large datasets into smaller chunks
  • Use overlapping window technique for continuous signals
  • Consider using CPU implementation for very large arrays

Performance Tips

  • Reuse Engines: Create FFTEngine once and reuse for multiple transforms
  • Dispose Resources: Always call dispose() when done to free GPU memory
  • Use CPU for Small Data: For transforms < 1024 elements, CPU may be faster
  • Batch Processing: Process multiple same-size transforms together

Still Having Issues?

If you're still experiencing problems:

  • Check the GitHub Issues page
  • Open a new issue with detailed information about your environment
  • Include browser version, OS, and a minimal reproduction example

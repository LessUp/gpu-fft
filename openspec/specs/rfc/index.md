## RFC Process

New architectural changes use OpenSpec workflow:

1. `/opsx:propose` — Create change proposal
2. `/opsx:apply` — Implement changes
3. `/opsx:archive` — Archive and update specs

## Governance Notes

- `openspec/specs/` is the canonical specification root for this repository
- The legacy top-level `/specs` tree has been removed; `openspec/specs/` is the only canonical spec root
